
# Lab 5: Post Office & Shipments

## Мета
Навчитися створювати узагальнені класи і методи (Generics), працювати з колекціями та LINQ, контролювати дані і обробляти винятки.

## Що реалізовано
- Сутності: Shipment і PostOffice (PostOffice містить колекцію відправлень).
- Узагальнений компонент Repository<T> з методом TopN для сортування і вибору топ-елементів.
  - Рекомендована сигнатура: IEnumerable<T> TopN(int n, Func<T, TKey> keySelector, bool desc = true)
  - Не використовувати зайву дженерикову змінну в сигнатурі методу.
- Власний виняток InvalidShipmentDatesException для помилок дат.
- Обчислення по колекціях: середній термін доставки, відсоток втрачених/пошкоджених відправлень, топ-напрямки.
- Демонстрація в Program.cs з обробкою винятків (try-catch).

## Контрольні питання
- Generics — узагальнення типів для безпечного і повторно використовуваного коду.  
- ArrayList vs List<T> — ArrayList не типізований; List<T> — типобезпечний і швидший.  
- Dictionary<TKey,TValue> — зберігає пари ключ-значення; List<T> — послідовний список.  
- finally — виконується завжди після try/catch для очищення ресурсів.  
- Власні виключення — для чіткої ідентифікації специфічних помилок.
