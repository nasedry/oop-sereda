# Lab 5: Post Office & Shipments

## Мета
Навчитися створювати узагальнені класи і методи (Generics), працювати з колекціями та LINQ, контролювати дані і обробляти винятки.

## Що реалізовано

- Узагальнений компонент Repository<T> з методом TopN для сортування і вибору топ-елементів.
  - Рекомендована сигнатура: IEnumerable<T> TopN(int n, Func<T, TKey> keySelector, bool desc = true)
  - Не використовувати зайву дженерикову змінну в сигнатурі методу.
- Власний виняток InvalidShipmentDatesException для помилок дат.
- Обчислення по колекціях: середній термін доставки, відсоток втрачених/пошкоджених відправлень, топ-напрямки.
- Демонстрація в Program.cs з обробкою винятків (try-catch).

## Контрольні питання

Що таке generics і які їх переваги
1.Generics — це типи, які працюють із будь-якими даними. Вони дозволяють писати універсальний код без копіювання для кожного типу і роблять його безпечнішим для типів.
2.Які основні відмінності між ArrayList і List<T>
ArrayList може зберігати будь-які об’єкти, але не перевіряє тип під час компіляції. List<T> типізований, безпечний для типів і працює швидше.
3.Чим відрізняється Dictionary<TKey,TValue> від List<T>
List<T> — це просто список елементів, шукає по індексу. Dictionary<TKey,TValue> зберігає пари ключ-значення і швидко шукає по ключу.
4.У чому перевага LINQ над класичними циклами
LINQ дозволяє писати компактний і легший для читання код при роботі з колекціями: фільтрувати, сортувати і групувати без ручних циклів.
5.Як працює ключове слово finally
finally виконується завжди після try-catch, навіть якщо був виняток. Зручно для закриття файлів, з’єднань і очищення ресурсів.
6.Коли доцільно створювати власні класи винятків
Коли хочеш чітко сказати, яка саме помилка сталася і обробити її окремо. Наприклад тут створено InvalidShipmentDatesException, щоб відловлювати помилки дат відправлення.
