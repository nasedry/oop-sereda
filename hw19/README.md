# Composite: дерево як єдина колекція

## Вступ

Патерн Composite (Компонувальник) використовується для роботи з ієрархічними структурами типу "частина-ціле".  
Його основна ідея полягає в тому, що як окремі об’єкти (Leaf), так і їхні групи (Composite) мають спільний інтерфейс.  
Завдяки цьому клієнтський код може працювати з усіма елементами однаково.

---

## Як Composite дозволяє працювати з деревом як з єдиною колекцією

1. Всі елементи реалізують спільний інтерфейс (Component).
2. Групи можуть містити інші компоненти.
3. Виклики методів передаються дочірнім елементам рекурсивно.
4. Клієнт не перевіряє тип об’єкта (Leaf чи Composite).

Таким чином, дерево об’єктів сприймається як одна структура.

---

## Приклади реального використання

### 1. Файлова система
Файл — це Leaf.  
Папка — це Composite.  
Метод підрахунку розміру може виконуватися для всієї структури незалежно від вкладеності.

### 2. Графічний редактор
Окремі фігури — Leaf.  
Група фігур — Composite.  
Можна переміщати або змінювати групу як один об’єкт.

### 3. Меню в програмі
Пункт меню — Leaf.  
Підменю — Composite.  
Кожен елемент має методи типу Render() або Click().

---

## Переваги

- Спрощує клієнтський код.
- Дозволяє працювати з ієрархією однаково.
- Легко розширювати систему новими компонентами.
- Добре підходить для рекурсивних структур.

---

## Недоліки

- Складніше обмежити типи компонентів.
- Інтерфейс може стати занадто загальним.
- Можлива надлишкова гнучкість.

---

## Висновок

Composite — це зручний патерн для роботи з деревоподібними структурами.  
Він дозволяє спростити код і працювати з ієрархією як з єдиною колекцією без перевірок типів.
